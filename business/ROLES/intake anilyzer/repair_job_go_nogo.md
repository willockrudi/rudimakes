# REPAIR JOB GO / NO-GO RUBRIC

Score each category 0–2 and total it.

| Category                 | 0                      | 1              | 2                 |
| ------------------------ | ---------------------- | -------------- | ----------------- |
| Item Value               | Very low               | Medium         | High              |
| Serviceability           | Hard / unknown         | Normal         | Easy / known      |
| Parts Availability       | Hard / obsolete        | Available      | In stock / common |
| Symptom Clarity          | Vague                  | Somewhat clear | Very clear        |
| Failure Type             | Intermittent / complex | Unknown        | Known/common      |
| Previous Repair          | Yes, messy             | Yes, unknown   | No                |
| Physical Condition       | Damaged                | Used           | Clean             |
| Economic Fit             | Likely not worth it    | Maybe          | Makes sense       |
| Customer Expectations    | Unrealistic            | Unknown        | Reasonable        |
| Your Experience w/ Model | Never seen             | Some           | Many times        |

### Scoring:

* **0–7 → NO GO** (Politely decline)
* **8–12 → CAUTION** (Diagnosis only, no estimate yet)
* **13–16 → OK JOB** (Normal intake)
* **17–20 → GOOD JOB** (Prioritize this job)

This alone will save you from nightmare jobs.

# ECONOMIC REALITY CHECK (Very Important)

Before accepting, ask:

1. What is this item worth working?
2. What is the customer likely willing to spend?
3. What is the worst-case bench time?
4. Could this turn into 5–10 hours?
5. Are parts available?
6. If I fix it, will something else fail next week?
7. Will this customer be happy if the bill is $___?
8. Would I personally pay to fix this if it were mine?

If the economics don’t make sense, **say so early**. Customers usually appreciate honesty **before** the repair, not after the bill.

# STANDARD SHOP INTAKE POSITIONS

You should mentally put every job into one of these categories:

| Category        | What it means                      |
| --------------- | ---------------------------------- |
| Standard Repair | Known issue, normal job            |
| Diagnosis Job   | Unknown fault, bench time required |
| Intermittent    | Open-ended bench time              |
| Restoration     | Time-heavy, parts-heavy            |
| Previous Repair | Risk of hidden problems            |
| Not Economical  | Warn customer early                |
| Parts-Dependent | Cannot quote until parts found     |
| Quick Fix       | Small known repair                 |
| High Risk       | Could become time sink             |
| Decline         | Not a good job for the shop        |

Your Intake AI should assign one of these every time.

# WHAT KILLS REPAIR SHOPS (So You Avoid It)

These are the jobs that destroy your hourly rate:

1. Intermittent problems
2. Previous failed repair attempts
3. Heavily modified gear
4. Vintage restorations with customer on a budget
5. Digital gear with unavailable parts
6. Cheap gear with labor-heavy repair
7. “It worked, then stopped, now it works again”
8. No schematics + complex PCB
9. Customer wants constant updates
10. Customer wants quote before diagnosis on complex issue

You don’t have to refuse all of these — but they should become:

> **Diagnosis jobs with clear bench rate and no hard estimate**

# STANDARD RULES MANY SHOPS USE

You may want policies like:

* Intermittent issues → diagnosis only
* Previous repair attempts → diagnosis only
* Liquid damage → no estimate until inspection
* Vintage restoration → hourly only
* No schematic → hourly only
* Digital board issues → no quote until parts availability confirmed
* Blows fuse → power section diagnosis first
* “No sound” → minimum bench fee
* Modded gear → no warranty
* Customer-supplied parts → no warranty
* Rush jobs → rush fee
* Abandoned gear → becomes shop property after X days

This protects you.

# SIMPLE DECISION TREE

You can literally follow this:

```
Do I know this model?
 ├─ No → Risk up
 └─ Yes → Continue

Is the problem clear?
 ├─ No → Diagnosis job
 └─ Yes → Continue

Has someone worked on it before?
 ├─ Yes → Diagnosis job
 └─ No → Continue

Is the problem intermittent?
 ├─ Yes → Diagnosis job
 └─ No → Continue

Are parts available?
 ├─ No → Parts-risk job
 └─ Yes → Continue

Does repair cost make sense vs value?
 ├─ No → Warn customer
 └─ Yes → Accept job
```

# The Real Goal

You are trying to sort every job into one of these **before it hits the bench**:

| Type            | How you charge         |
| --------------- | ---------------------- |
| Quick Fix       | Flat or minimum        |
| Standard Repair | Estimate               |
| Diagnosis       | Bench hourly           |
| Intermittent    | Bench hourly           |
| Restoration     | Bench hourly           |
| Previous Repair | Bench hourly           |
| Digital Board   | Bench + parts research |
| Not Economical  | Warn / Decline         |

If you do just this, your business will run **way smoother** and your effective hourly rate will go up without raising prices — because you stop taking the wrong jobs.
